# v2.2.4 - Version Updates & Stability

**Release Date:** December 13, 2025  
**Version Range:** 2.2.2 - 2.2.4

## Summary

This release series focuses on version management improvements and stability fixes following the major review flow refactor in v2.2.1. The key change is dynamic version reading from package.json, eliminating the need to update hardcoded version strings during releases.

## Key Changes

### Dynamic Version Reading (v2.2.4)
- Version now read dynamically from `package.json` at runtime
- Eliminates version mismatch issues between package.json and CLI output
- No more forgetting to update hardcoded version strings during releases
- Improves release workflow reliability

### Version Display Fixes (v2.2.3)
- Fixed hardcoded version references that were out of sync
- Ensures consistent version display across all CLI output

### npm Publishing (v2.2.2)
- Version bump for npm publishing
- No functional changes

## Technical Details

### Before (Hardcoded)
```typescript
// Had to update manually on each release
const VERSION = "2.2.3";
```

### After (Dynamic)
```typescript
// Reads from package.json automatically
import pkg from "../package.json";
const VERSION = pkg.version;
```
